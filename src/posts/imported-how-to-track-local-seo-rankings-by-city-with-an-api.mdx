<p>Local SEO rankings are not the same everywhere.</p>

<p>Your website may rank in position 2 in Austin, position 8 in Dallas, and not appear at all in Chicago.</p>

<p>That is why checking one generic Google result is not enough for local SEO.</p>

<p>If you are working on <a href="https://www.talordata.com/serp-api/seo-monitoring" rel="noopener noreferrer">local SEO</a>, agency reporting, competitor monitoring, or location-based search analysis, you need to track rankings by city.</p>

<p>A simple workflow looks like this:<br>
</p>

<div class="highlight js-code-highlight">
<pre class="highlight plaintext"><code>Keyword + city → SERP API → local search results → ranking check → CSV report
</code></pre>

</div>



<p>In this tutorial, we’ll build a basic Python script that tracks local SEO rankings by city using a SERP API.</p>

<p>We will:</p>

<ol>
<li>Define target keywords</li>
<li>Define target cities</li>
<li>Send city-specific searches to a SERP API</li>
<li>Extract organic results</li>
<li>Check where a target domain appears</li>
<li>Save the ranking data to CSV</li>
</ol>

<p>This is not a full SEO platform, but it gives you the core logic behind many local rank tracking tools.</p>

<h2>
  
  
  Why city-level rankings matter
</h2>

<p>Google search results are location-sensitive.</p>

<p>A query like:<br>
</p>

<div class="highlight js-code-highlight">
<pre class="highlight plaintext"><code>best digital marketing agency
</code></pre>

</div>



<p>may return different results in:<br>
</p>

<div class="highlight js-code-highlight">
<pre class="highlight plaintext"><code>New York
Austin
London
Singapore
Sydney
</code></pre>

</div>



<p>This matters even more for local intent keywords, such as:<br>
</p>

<div class="highlight js-code-highlight">
<pre class="highlight plaintext"><code>dentist near me
plumber in Chicago
coffee shop in Austin
real estate agent in Miami
</code></pre>

</div>



<p>For these searches, the result page can include:</p>

<ul>
<li>organic results</li>
<li>local packs</li>
<li><a href="https://docs.talordata.com/serp-api/query-parameters/google/google-maps" rel="noopener noreferrer">Google Maps results</a></li>
<li>ads</li>
<li>business directories</li>
<li>review sites</li>
<li>service pages</li>
<li>location-specific landing pages</li>
</ul>

<p>If you only check one location, you may miss what users actually see in other cities.</p>

<p>For local SEO, ranking data without location context is incomplete.</p>

<h2>
  
  
  Why use a SERP API?
</h2>

<p>You could try to check Google rankings manually.</p>

<p>But that does not scale.</p>

<p>You could also try to scrape Google directly, but that brings a lot of maintenance work:</p>

<ul>
<li>changing page layouts</li>
<li>CAPTCHA</li>
<li>blocked requests</li>
<li>proxy handling</li>
<li>inconsistent HTML</li>
<li>location mismatch</li>
<li>parser updates</li>
<li>retry logic</li>
</ul>

<p>A <a href="https://www.talordata.com/products/serp-api" rel="noopener noreferrer">SERP API</a> gives you structured search results in JSON.</p>

<p>Instead of parsing raw HTML, you get data like this:<br>
</p>
