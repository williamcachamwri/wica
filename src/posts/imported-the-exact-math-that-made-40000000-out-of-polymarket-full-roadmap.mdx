<h2>
  
  
  Why Simple "YES + NO = 1" Checks Fail
</h2>

<p>Most retail traders stop at basic price sum checks. That's not enough.</p>

<p>Markets are logically dependent. Example:</p>

<ul>
<li>"Will Trump win Pennsylvania?" → YES: $0.48</li>
<li>"Will Republicans win Pennsylvania by 5+ points?" → YES: $0.32</li>
</ul>

<p>If the second outcome happens, the first <em>must</em> be true. These dependencies create arbitrage opportunities that simple addition cannot detect.</p>

<p>This is known as the <strong>marginal polytope problem</strong> — projecting prices onto the set of arbitrage-free probability distributions.</p>

<h2>

<h2>
  
  
  The Core Math: Bregman Projection + Frank-Wolfe
</h2>

<p>To find the <em>optimal</em> trade, you project the current market prices onto the nearest arbitrage-free probability distribution using <strong>Bregman divergence</strong> (logarithmic distance that respects probability structure).</p>

<p>Direct projection is intractable, so the <strong>Frank-Wolfe algorithm</strong> is used:</p>

<ul>
<li>Start with a small active set of valid outcomes</li>
<li>Iteratively solve linear programs</li>
<li>Add one new vertex per iteration</li>
<li>Converges in 50–150 iterations instead of exploring 2ⁿ space</li>
</ul>

<p>As more outcomes are resolved (e.g., games completed), the feasible set shrinks dramatically and solves become faster — from 10–30 seconds early in an event to under 5 seconds near the end.</p>
