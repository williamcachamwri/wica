<p>Every time I used Opus Clip or Vidyo.ai, the same thought hit me:<br>
I’m paying $20/month to upload my video to someone else’s server,<br>
wait in a queue, and hope their AI finds something useful.</p>

<p>So I built an alternative that runs entirely in the browser.<br>
No file uploads. No subscriptions. No server costs on my end.<br>
The result is ClipGG’s AI Video Highlights tool —<br>
and in this post I’ll walk through exactly how it works technically.</p>


<h2>
  
  
  The core problem I was solving
</h2>

<p>Finding highlights in a long video is genuinely hard to automate well.<br>
The expensive approach: transcribe with Whisper, feed text to GPT-4,<br>
profit. But that requires a backend, API costs, and user uploads.</p>

<p>I wanted zero server involvement.<br>
That meant doing everything with browser APIs.</p>


<h2>
  
  
  What actually runs in the browser
</h2>

<p>The pipeline has four stages:</p>
<h3>
  
  
  1. File reading — no upload needed
</h3>

<h3>
  
  
  2. Audio analysis — Web Audio API + Web Worker
</h3>

<p>I use <code>OfflineAudioContext</code> to decode audio faster than real-time,<br>
then downsample to 8000–11025 Hz before analysis.<br>
This reduces RAM usage from ~115MB to ~19MB for a 10-minute video.<br>
</p>
